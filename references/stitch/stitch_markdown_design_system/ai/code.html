AI Engineering Workflow

Scaling efficiency through model-augmented software engineering. A systematic approach to transforming complex problems into precise, scalable solutions using state-of-the-art AI tools at every stage of the development lifecycle.

01 / DISCOVERY
science

调研

市场分析与可行性评估，以验证技术路径。

输入: 问题陈述 输出: 调研笔记
Perplexity GPT-4o

integration_instructions 应用案例: Job Tracker 岗位分析

使用 Perplexity 收集求职者痛点，确认 JD 解析自动化的市场需求和可用的解析模型 (如 Claude/GPT)。

产出: 综合调研报告
02 / DEFINITION
description

需求定义

定义用户价值、核心功能和成功指标。

输入: 调研笔记 输出: Markdown 规范
Claude 3.5

integration_instructions 应用案例: 个人简约记账本 APP

通过 Claude 3.5 提炼极简记账的核心要素，生成包含“一键记账”、“月度趋势分析”等功能的详细 Markdown PRD。

产出: Markdown 需求文档
03 / DESIGN
architecture

架构设计

系统设计、组件映射和数据流建模。

输入: Markdown 规范 输出: 架构图
Mermaid.js Diagrams

integration_instructions 应用案例: Job Tracker + JD Analyzer

使用 Mermaid.js 绘制架构图，定义从 Chrome 插件抓取 JD 到后端 AI 解析器处理，再到前端看板展示的数据流。

产出: 架构图表
04 / PLANNING
checklist

任务拆解

将架构组件分解为可执行的原子任务。

输入: 架构图 输出: 任务队列
Linear GitHub Issues

integration_instructions 应用案例: 个人简约记账本 APP

在 GitHub Issues 中建立原子任务库：如"实现数字小键盘组件"、"设计本地 SQLite 数据库表结构"等。

产出: 原子任务卡片
05 / IMPLEMENTATION
terminal

代码实现

高速、AI辅助的代码编写和逻辑实现。

输入: 任务队列 输出: 源代码
Cursor Copilot

integration_instructions 应用案例: 个人简约记账本 APP

使用 Cursor/Copilot 辅助编写记账本 APP 的核心账单统计与分类处理逻辑，加速迭代开发过程。

产出: 源代码库
06 / VERIFICATION
bug_report

测试验证

严格的质量保证和测试覆盖率验证。

输入: 源代码 输出: 测试报告
Jest Pytest

integration_instructions 应用案例: 个人简约记账本 APP

通过 Jest 编写测试用例，验证金额计算逻辑的准确性，确保在各种边缘情况下（如大额输入）不会崩溃。

产出: 自动化测试报告
07 / VALIDATION
fact_check

验收交付

对照原始 PRD 标准进行人工在环验证。

输入: 测试报告 输出: 验收通过
人工审核

integration_instructions 应用案例: Job Tracker + JD Analyzer

人工测试 JD 解析器提取薪资、技能要求的准确率，确认 UI 操作符合初始定义的效率提升目标。

产出: 演示 / 签名代码
08 / EVOLUTION
loop

持续迭代

基于实际使用情况和指标进行持续改进。

输入: 实时指标 输出: 新需求
数据分析

integration_instructions 应用案例: 个人简约记账本 APP

根据初期用户的反馈收集，增加“订阅制账单自动识别”功能，开启新一轮的开发循环。

产出: 反馈循环数据
